1. EXECUTIVE SUMMARY
================================================================================

  8 specialized code analysis agents reviewed all 171 C# source files and
  ~35 Lua behavior scripts across every major system in the SizeBox codebase.

  TOTAL ISSUES FOUND: ~310
  - Critical:  ~35
  - High:      ~75
  - Medium:    ~120
  - Low:       ~80

  OVERALL CODE QUALITY SCORE: 4/10

  The codebase is a functional but fragile Unity 2017.3 game with significant
  technical debt. The most pervasive problems are:

  1. Missing null checks causing NullReferenceExceptions
  2. Scale-dependent calculations that break at extreme sizes
  3. Network synchronization gaps (UNet)
  4. Memory leaks (meshes, textures, event listeners)
  5. Frame-rate-dependent physics and movement
  6. Lua scripting API case-sensitivity mismatches
  7. Dead/unreachable code paths

  The game likely "works" at normal play because many bugs only manifest at
  extreme scales, during multiplayer, or in edge-case interactions. A new
  maintainer should expect frequent NullReferenceExceptions and occasional
  crashes during extended play sessions.

3. CRITICAL FIXES (Must Fix -- Crashes & Data Corruption)
================================================================================

--- CRASH BUGS ---

C-01: GRAVITY NULL REFERENCE -- CRASH EVERY PHYSICS FRAME
  File: Agents/Gravity.cs:15
  When GetComponent<Rigidbody>() returns null, AddComponent is called but
  the result is NOT assigned back to rbody. rbody stays null. FixedUpdate
  then calls rbody.AddForce() every frame = NullReferenceException.
  FIX: Change to "rbody = gameObject.AddComponent<Rigidbody>();"

C-02: JIGGLEBONE INVALID QUATERNION -- NaN PROPAGATION
  File: JiggleBone.cs:39
  "transform.rotation = new Quaternion()" creates quaternion (0,0,0,0)
  which is INVALID. Identity is (0,0,0,1). This causes NaN propagation
  through all child bone transforms, corrupting the entire skeleton.
  FIX: Use "transform.rotation = Quaternion.identity;"

C-03: GIANTESS IK FOOT COLLIDER NULL CRASH
  File: GtsScripts/IK/GiantessIK.cs:194-199
  leftFootCollider is searched by name prefix "Coll". If no matching
  child exists, it stays null. Lines 194-199 access .transform and
  .sharedMesh unconditionally = NullReferenceException on spawn.
  FIX: Add null check before accessing leftFootCollider.

C-04: MICRO COLLISION NULL DEREF IN PARENT TRAVERSAL
  File: Agents/Micro.cs:61-63
  While loop sets t = null to break, then line 63 does t = t.parent
  which dereferences null = NullReferenceException.
  FIX: Use "break;" instead of "t = null;"

C-05: A* PATHFINDING INFINITE LOOP
  File: AI/Pathfinding/AStar.cs:126-129
  "while(current.node != start)" walks back through previous pointers.
  If closestNodeToGoal's chain doesn't include start, this is an
  infinite loop or null deref when current.previous is null.
  FIX: Add guard: "while(current.previous != null && current.node != start)"

C-06: LUA MATHF.PERLINNOISE INFINITE RECURSION
  File: Lua/LuaBinding/LuaMathf.cs:316
  "return PerlinNoise(x,y)" calls itself instead of
  UnityEngine.Mathf.PerlinNoise(x,y) = StackOverflowException.
  FIX: Change to "return UnityEngine.Mathf.PerlinNoise(x, y);"

C-07: BEHAVIOR QUEUE CRASHES ON LUA CREATE FAILURE
  File: AI/Behavior.cs:58 + AI/BehaviorQueue.cs:137
  Behavior.CreateInstance returns null on Lua exception. BehaviorQueue
  immediately calls instance.Start() on null = NullReferenceException.
  FIX: Null-check CreateInstance return before calling Start.

C-08: AUDIO LOADER RETURNS UNLOADED CLIP
  File: RuntimeLoader/AudioLoader.cs:46-48
  WWW request created but audioClip accessed immediately without
  waiting for download. Returns empty/corrupted clip every time.
  FIX: Use coroutine with yield or UnityWebRequestMultimedia.

C-09: AUDIO LOADER NULL REFERENCE ON MISSING CLIP
  File: RuntimeLoader/AudioLoader.cs:12-20
  TryGetValue fails, logs error, then accesses clip.name on null clip.
  FIX: Return null after logging error (add early return).

C-10: IOMANAGER COPY-PASTE BUG LOADS WRONG ASSET
  File: RuntimeLoader/IOManager.cs:370
  "vertexBytes = bundle.LoadAsset(modelName + ".index.bytes")" should
  be ".vertex.bytes". Loads index data as vertex data = corrupted models.
  FIX: Change to ".vertex.bytes".

--- DATA CORRUPTION ---

C-11: PREFERENCES NEVER ACTUALLY SAVE
  File: PreferencesCentral.cs:71 and 104
  FloatStored.value and BoolStored.value setters use the property name
  "value" which shadows the C# setter keyword. The assignment "val = value"
  either no-ops or recurses infinitely. User preferences are never persisted.
  FIX: Rename the property to avoid shadowing the setter keyword.

C-12: CREATE MESH MEMORY LEAK -- UNBOUNDED GPU MEMORY
  File: GtsScripts/CreateMesh.cs:29-36
  UpdateCollider creates new Mesh() every call, assigns to sharedMesh.
  Old mesh is set to null but never Destroy()ed. Unity Meshes are native
  objects requiring explicit destruction. GPU memory grows without bound.
  FIX: Call Destroy() on old sharedMesh before creating new one.

C-13: IOMANAGER RENDER TEXTURE LEAK
  File: RuntimeLoader/IOManager.cs:744-775
  RenderTexture.GetTemporary() called but ReleaseTemporary() never called.
  Texture2D created but never destroyed. Called per-material during loading.
  FIX: Add RenderTexture.ReleaseTemporary(rt) and Destroy(img2).

C-14: SELECTDECISION CAN BE NULL (FLOATING POINT EDGE CASE)
  File: AI/GTSDecisionMaker.cs:44-58 + AI/MicroDecisionMaker.cs:45-57
  Weighted random selection: if choiceValue == totalScore exactly (float
  edge case), accumuledScore never exceeds it, selectDecision stays null.
  Line 57 crashes on selectDecision.behavior.
  FIX: Add fallback: if(selectDecision == null) selectDecision = decisions[decisions.Count - 1];

C-15: MICRODODECISIONMAKER CRASH ON FIRST EXECUTE
  File: AI/MicroDecisionMaker.cs:26
  Field "lastDesicion" never initialized. First Execute() when not idle
  accesses lastDesicion.behavior.react = NullReferenceException.
  FIX: Initialize lastDesicion or add null check.

4. HIGH SEVERITY FIXES (Gameplay-Breaking)
================================================================================

--- MOVEMENT & PHYSICS ---

H-01: CHARACTERS WALK THROUGH WALLS
  File: Agents/MovementCharacter.cs:111
  "rbody.position = displacement" teleports rigidbody, completely
  bypassing Unity collision detection. All AI characters can walk
  through walls, other characters, and all colliders.
  FIX: Use rbody.MovePosition(displacement).

H-02: GRAVITY DOESN'T SCALE WITH MASS
  File: Agents/Gravity.cs:24
  AddForce uses default ForceMode.Force which divides by mass. Small
  characters plummet instantly, large characters float languidly.
  FIX: Use ForceMode.Acceleration for consistent gravity feel.

H-03: STEERING COMPUTED IN WRONG UPDATE LOOP
  File: Agents/MovementCharacter.cs:89-199
  Steering computed in LateUpdate, movement applied in FixedUpdate.
  Causes jittering at low framerates and framerate-dependent speed.
  FIX: Move steering computation to FixedUpdate.

H-04: JUMP KILLS HORIZONTAL MOMENTUM
  File: PlayerScripts/Player.cs:428
  "rbody.velocity = new Vector3(0, jumpHeight * Scale, 0)" zeroes X/Z.
  Player stops moving forward mid-air on every jump.
  FIX: Preserve horizontal velocity during jump.

H-05: OBSTACLE DETECTOR IS A NO-OP STUB
  File: Agents/ObstacleDetector.cs:5-18
  CheckObstacle() always returns false. Wall avoidance does nothing.
  All AI characters have zero obstacle avoidance capability.
  FIX: Implement raycast-based detection or verify subclass exists.

H-06: PLAYER FPS CAMERA PERMANENTLY DISABLED
  File: PlayerScripts/Player.cs:472
  "if(climbing || true) return;" -- always returns. First-person camera
  rotation is completely broken. All code below is unreachable.
  FIX: Remove "|| true".

--- IK & ANIMATION ---

H-07: HAND GRAB NEVER RELEASES TARGET
  File: GtsScripts/IK/HandIK.cs:105-121
  Target is parented to hand bone during grab. Return state never
  unparents (SetParent(null)) or unlocks the target. Grabbed entities
  remain attached to the giantess hand permanently.
  FIX: Add target.transform.SetParent(null) and target.Unlock() in Return.

H-08: STOMP ACTION OPERATOR PRECEDENCE BUG
  File: AI/Actions/Interaction/StompAction.cs:18
  "hasStarted && agent.ik.crushEnded || target == null || target.isDead"
  evaluates as (hasStarted && crushEnded) || targetNull || targetDead.
  Reports completed before action starts if target is null/dead.
  FIX: Add parentheses for intended grouping.

H-09: BE ACTION INTERRUPT STARTS INSTEAD OF STOPS
  File: AI/Actions/Resizing/BEAction.cs:39
  Interrupt() sets BESpeed = speed (original speed) instead of 0.
  Interrupting breast expansion RESTARTS it instead of stopping.
  FIX: Change to giantess.BESpeed = 0f;

H-10: STATIC COLLIDERS ENABLE ALWAYS DESTROYS MESHES
  File: GtsScripts/Giantess.cs:322-329
  StaticCollidersEnable(bool option) ignores the parameter. Always
  calls DestroyMesh() regardless of true/false.
  FIX: Only DestroyMesh when option is false.

H-11: HUMANOID SETS GIANTESS ANIMATOR ON ALL HUMANOIDS
  File: Agents/Humanoid.cs:53
  SetPoseMode(false) applies giantess animator to ALL humanoids
  including micros and players. Wrong animation controller.
  FIX: Store and restore original animator controller.

--- NETWORKING ---

H-12: ENTITY SCALE SYNC BROKEN
  File: Agents/EntityBase.cs:158-165
  _ChangeScale called locally for all clients BEFORE network command.
  Clients without authority change scale before server validation.

H-13: MICRO PARENTING NOT NETWORKED

H-14: NPC CRUSH NOT NETWORKED

H-18: MULTIPLE LUA SCRIPTS BROKEN BY CASE SENSITIVITY
  Files: follow.lua, gts_eat.lua, gts_grab.lua, gts_shrink_stomp.lua,

  File: EventManager.cs:12-43

H-24: SUN CONTROLLER CAPTURES ALL INPUT GLOBALLY

7. ARCHITECTURE-WIDE PATTERNS
================================================================================

  PATTERN 1: MISSING NULL CHECKS
  Nearly every GetComponent, Camera.main, singleton.Instance, and bone
  transform lookup lacks null protection. This is the #1 source of crashes.
  SCOPE: ~50+ instances across all systems.

  PATTERN 2: SCALE-DEPENDENT HARDCODING
  Magic numbers (850, 200, 1200, 400, 2000, etc.) are used throughout the
  IK, collision, and physics systems. None scale with entity size. The game
  was tuned for a specific scale range and breaks outside it.
  SCOPE: ~30+ instances in GtsScripts/, Agents/, Camera/.

  PATTERN 3: SINGLETON FRAGILITY
  GameController, MainController, ObjectManager, IOManager, StateManager,
  NPCSpawner all use fragile singleton patterns with no duplicate protection,
  inconsistent initialization timing, and missing DontDestroyOnLoad.
  SCOPE: 6+ singletons.

  PATTERN 4: EVENT LISTENER LEAKS
  EventManager has no unregister method. Every registered listener stays
  forever. SoundManager, PlayerCamera, and others register but never
  unregister, causing null callbacks after scene transitions.
  SCOPE: All event listeners.

  PATTERN 5: FRAME-RATE DEPENDENT BEHAVIOR
  Steering in LateUpdate, movement in FixedUpdate. Lerp with deltaTime
  (asymptotic, never converges). JiggleBone physics with no time step.
  SCOPE: MovementCharacter, JiggleBone, HeadIK, GiantessIK.

  PATTERN 6: STATIC SHARED STATE
  PathNode.connections, WorldNodes.nodeArray, CityBuilding statics,
  ResizeCharacter.minSize/maxSize, CenterOrigin.aux -- all use static
  fields that are shared across instances and can cause data corruption.
  SCOPE: 6+ classes.
